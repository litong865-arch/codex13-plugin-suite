# 安装说明-小白版

## 你要完成什么

把 13 个 Codex 本地插件安装到你自己的 Codex 里，让你能从“已安装插件”列表中直接选择使用。

## 安装前准备

你只需要：

- Windows 电脑。
- 已经安装 Codex。
- 这个压缩包已经下载到电脑上。

不需要管理员权限。安装内容只会写入当前用户目录：

- `%USERPROFILE%\.codex\plugins`
- `%USERPROFILE%\.agents\plugins\marketplace.json`
- `%USERPROFILE%\.codex\config.toml`

## 第一步：解压

1. 找到压缩包。
2. 右键压缩包。
3. 点击“全部解压缩”。
4. 解压到桌面或任意你能找到的位置。

不要直接在压缩包里面运行安装脚本，必须先解压。

## 第二步：双击安装

1. 打开解压后的文件夹。
2. 找到 `Install-Double-Click.bat`。
3. 双击它。
4. 如果 Windows 弹出安全提示，确认这是你信任的本地插件包后继续运行。
5. 等 PowerShell 窗口显示“安装完成”。

## 第三步：重启 Codex

安装完成后，必须完全退出并重启 Codex。

原因：Codex 的插件和技能通常在新会话或重启后才会重新加载。

## 第四步：检查是否成功

重启 Codex 后：

1. 新开一个会话。
2. 打开插件列表。
3. 查看已安装插件。
4. 找到 `项目总控闸门 / project-master-gate`。
5. 选择它。
6. 输入测试提示词：

```text
先不要写代码，请使用项目总控闸门帮我分析这个项目：

我想做一个餐饮商家 AI 评论回复 Agent。

请告诉我应该按什么插件顺序推进，并给出每一步的调用提示词。
```

## 如果双击安装失败

可以手动打开 PowerShell：

1. 进入解压后的文件夹。
2. 在空白处按住 Shift，点击鼠标右键。
3. 选择“在终端中打开”或“在 PowerShell 中打开”。
4. 运行：

```powershell
Set-ExecutionPolicy -Scope Process Bypass -Force
.\install.ps1
```

## 安装脚本做了什么

安装脚本会：

1. 检查 13 个插件目录是否存在。
2. 复制插件到 `%USERPROFILE%\.codex\plugins`。
3. 如果发现同名旧插件，会先备份到 `%USERPROFILE%\.codex\plugins\backups`。
4. 更新个人 marketplace。
5. 写入 Codex 已启用插件配置。
6. 同步 installed cache。

它不会删除其他无关插件。

## 卸载方式

如果以后想手动卸载，可以删除这些位置里的对应插件：

- `%USERPROFILE%\.codex\plugins\<plugin-id>`
- `%USERPROFILE%\.codex\plugins\cache\personal\<plugin-id>`

然后从 `%USERPROFILE%\.codex\config.toml` 中删除对应的 `[plugins."<plugin-id>@personal"]` 配置段。
