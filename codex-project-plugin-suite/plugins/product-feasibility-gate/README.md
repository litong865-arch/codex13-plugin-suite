# 产品可行性闸门 V3.1

版本：3.1.0

这是一个 Codex 开发前置闸门插件。

它的作用是在 Codex 开发项目之前，先完成：

- Idea 阶段：目标用户、真实痛点、商业假设、竞争替代
- MVP 阶段：主线流程、范围控制、API / Mock、数据结构
- 开工准备层：docs / app / design、PRD 与 Spec、Git、角色分工
- AI 工程脚手架层：分层架构、知识地图、反馈系统、静态检查、代码漂移、持续重构
- Launch 阶段：PMF 指标、反馈闭环
- Scale 阶段：成本、限流、数据飞轮、平台依赖、护城河
- 验收标准
- 异常处理
- 安全与隐私
- Codex Goal 指令生成

## 新增自视频 A：造脚手架：工程师的新角色

新增能力：

- AI 工程脚手架检查
- 分层架构边界
- AI 可读知识地图
- 反馈系统
- 静态检查
- Code Drift 检查
- 持续清理与重构

## 新增自视频 B：Vibe Coding 开工前准备

新增能力：

- 开工前目录结构
- PRD 与 Spec 区分
- Spec 最低要求
- Git 初始化和回滚点
- AI 角色分工

## 适合场景

- AI Agent
- 小程序
- APP
- SaaS
- 自动化系统
- 餐饮评论回复工具
- 周易玄学 APP
- 短视频自动化工具
- AI 图片 / 视频工具
- Codex 插件开发
- 任何商业化产品想法
- 任何需要 Codex 长期开发的项目

## 使用方式

在 Codex 中输入：

使用 product-feasibility-gate 帮我检查这个项目，先不要写代码。

或者：

先走产品可行性闸门 V3.1，帮我判断这个项目值不值得做、MVP 主线怎么跑通、开工准备是否完整、AI 工程脚手架是否可靠。
