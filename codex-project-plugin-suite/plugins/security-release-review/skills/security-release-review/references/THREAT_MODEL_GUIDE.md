# Threat Model Guide

威胁建模必须覆盖：

1. 核心资产：账号、隐私数据、后台、API key、AI/SMS 成本、文件存储、支付、订单、内容数据。
2. 攻击者：游客、注册用户、恶意用户、脚本、广告党、盗号者、内部误操作、prompt injection 攻击者、成本攻击者。
3. 数据流：输入来源、存储位置、公开展示位置、进入 AI 的内容、产生成本的接口、能修改数据的接口。
4. 信任边界：浏览器/后端、后端/数据库、后端/第三方服务、后台管理员/普通用户。
5. Top 10 项目风险：攻击方式、影响、证据、建议、是否阻断上线。

威胁模型不是泛泛而谈，必须引用具体文件、API、功能或 finding。
