# AI Agent Security Guide

AI / Agent 安全重点：

- 系统提示词、内部规则、API key 必须在服务端。
- 用户输入不能直接控制工具名称、文件路径、SQL、URL、短信目标、支付目标。
- 工具必须有 allowlist、参数校验、权限边界和审计日志。
- 高危工具需要人工确认：删文件、写数据库、发短信、发邮件、扣费、改管理员权限、访问本地文件、执行 shell。
- 必须测试 prompt injection、系统提示词泄露、API key 泄露、越权工具调用、高 token 消耗、敏感输出、不安全文件访问。

不要用“提示词要求模型不要做坏事”替代服务端权限、限流、审核和审计。
