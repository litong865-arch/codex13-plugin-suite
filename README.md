# Codex 13 插件项目流

这是一套面向技术小白的 Codex 项目开发插件包。它的目标不是让小白立刻变成程序员，而是把一个项目从想法、规格、开发、验收、上线到商业验证，拆成能照着问、照着检查、照着交付的流程。

核心思路：先总控，再分派；先检查，再开发；先验收，再交付。

## 适合谁

- 想用 Codex 做 AI Agent、SaaS、自动化工具、网页应用、小程序原型的人
- 不熟悉完整软件开发流程，但想把项目做完整的技术小白
- 已经多轮让 AI 写代码，项目开始变乱、变难验收的人
- 需要把项目交给别人安装、运行、演示或继续维护的人

## 小白安装方式

1. 下载 `release/Codex项目开发插件13件套安装包.zip`。
2. 右键压缩包，选择“全部解压”。
3. 打开解压后的 `codex-project-plugin-suite` 文件夹。
4. 双击 `Install-Double-Click.bat`。
5. 看到安装完成提示后，关闭窗口并重启 Codex。
6. 重启后进入已安装插件列表，优先选择“项目总控闸门”开始使用。

如果你是从 GitHub 的源码包下载，也可以进入 `codex-project-plugin-suite/` 文件夹后运行同样的安装脚本。

## 13 个插件速查

| 中文名称 | 插件 ID | 用途 |
| --- | --- | --- |
| 项目总控闸门 | `project-master-gate` | 判断项目阶段，分派插件、工具和子线程。 |
| 决策压力测试器 | `decision-stress-test-gate` | 判断项目值不值得做，找出致命假设。 |
| 产品可行性闸门 | `product-feasibility-gate` | 确认 MVP 主线、功能边界和验收标准。 |
| 安全闸门 | `security-release-review` | 检查安全、隐私、密钥、权限和上线阻断项。 |
| 规格生成器 | `spec-builder` | 把模糊想法整理成 Codex 可执行规格。 |
| 接口/平台审查器 | `api-platform-gate` | 检查平台 API、权限、规则、风控和替代方案。 |
| AI 成本估算器 | `ai-cost-estimator` | 估算模型/API 成本、延迟、额度、缓存和降级。 |
| 代码漂移巡检器 | `code-drift-inspector` | 检查多轮修改后的架构漂移和代码混乱。 |
| Codex 执行验收器 | `codex-acceptance-reviewer` | 验证功能是否真的完成，防止假完成。 |
| 上线发布守护器 | `release-guardian` | 检查构建、测试、环境变量、回滚和交付风险。 |
| 小白执行手册生成器 | `beginner-runbook-builder` | 生成小白能照做的运行、部署、排错手册。 |
| 用户访谈生成器 | `user-interview-generator` | 生成访谈问题、邀约话术、问卷和验证计划。 |
| 竞品分析器 | `competitor-analyzer` | 分析替代方案、差异化、切换理由和定价。 |

## 推荐使用流程

1. 先把项目想法用一句话说清楚。
2. 问 Codex：“我是技术小白，请先用项目总控闸门判断我现在该走哪一步，不要直接写代码。”
3. 按项目总控闸门输出的顺序调用对应插件。
4. 规格、接口、成本、安全这些前置问题清楚后，再让 Codex 写代码。
5. Codex 说完成后，用“代码漂移巡检器”和“Codex 执行验收器”检查。
6. 准备交付前，用“上线发布守护器”和“小白执行手册生成器”收尾。
7. 如果要商业化，再用“用户访谈生成器”和“竞品分析器”验证真实需求。

## 可直接复制给 Codex 的提示词

```text
我是技术小白，想做一个【在这里写你的项目】。
请不要直接写代码。先使用项目总控闸门判断：
1. 我现在处于项目哪个阶段？
2. 现在最应该调用哪几个插件？
3. 哪些问题如果不解决，会导致项目烂尾？
4. 请给我下一步可复制的提示词。
```

## 安全说明

这个仓库包含“安全闸门”插件，因此源码和模板里会出现 `token`、`secret`、`api key` 等安全审查关键词。这些是用于提醒用户检查风险的说明文本，不是实际密钥。发布前仍建议你用 GitHub secret scanning 或其他工具再扫一遍。
